# Walk: split `_cv_walk_*` into root-anchored halves

## Goal

Factor `_cv_walk_dim` / `_cv_walk_pos` / `_cv_walk_rect` in `src/geom.hc`
into two simpler halves anchored on the root layer, and reuse them from
`_root_pos` / `_root_rect` in the VS section.

## Naming

| name           | reading                       | direction      |
|----------------|-------------------------------|----------------|
| `_pos_root_to` | L_to is root (target = root)  | L_fr → root    |
| `_pos_root_fr` | L_fr is root (source = root)  | root → L_to    |
| `_dim_root_to` | (same)                        | L_fr → root    |
| `_dim_root_fr` | (same)                        | root → L_to    |
| `_rect_root_to`| (same)                        | L_fr → root    |
| `_rect_root_fr`| (same)                        | root → L_to    |

Reads naturally as "to-root" / "from-root" too.

## Signatures (sketch)

Each `_xxx_root_to` consumes a `Pico_Rel_*` in `L_fr`'s frame, walks up,
returns the corresponding `SDL_F*` in the root's frame plus the root layer.

Each `_xxx_root_fr` consumes the `SDL_F*` in root's frame and packs into
`L_to`'s frame.

```c
static Pico_Layer* _pos_root_to (Pico_Layer* L_fr, const Pico_Rel_Pos* fr,
                                 SDL_FPoint* out);
static void        _pos_root_fr (Pico_Layer* R,    SDL_FPoint p,
                                 Pico_Layer* L_to, Pico_Rel_Pos* to);
```

Same shape for `_dim_*` (with `SDL_FDim`) and `_rect_*` (with `SDL_FRect`).

## New `_cv_walk_*` body

```c
static void _cv_walk_pos (
    Pico_Layer* L_to, Pico_Rel_Pos* to,
    Pico_Layer* L_fr, const Pico_Rel_Pos* fr
) {
    if (L_fr == L_to) {
        Pico_Abs_Rect base = {0, 0, L_fr->scene.dim.w, L_fr->scene.dim.h};
        SDL_FPoint p = _sdl_pos(*fr, &base);
        _rel_pos(p, to, &base);
        return;
    }
    SDL_FPoint p;
    Pico_Layer* R = _pos_root_to(L_fr, fr, &p);
    pico_assert(R == _root_of(L_to) && "cv: layers must share a root");
    _pos_root_fr(R, p, L_to, to);
}
```

Same shape for `_cv_walk_dim` / `_cv_walk_rect`.

## VS rewire

`_root_pos` / `_root_rect` currently call `pico_cv_pos` / `pico_cv_rect`
with the root as `L_to`, which builds a 1-entry chain and runs a no-op
down-loop.
Replace with direct `_pos_root_to` / `_rect_root_to` calls.

```c
static Pico_Rel_Pos _root_pos (const char* layer, const Pico_Rel_Pos* p) {
    Pico_Layer* L = (layer == NULL) ? G.layer : _pico_layer_name(layer);
    SDL_FPoint pt;
    Pico_Layer* R = _pos_root_to(L, p, &pt);
    Pico_Abs_Rect Rb = {0, 0, R->scene.dim.w, R->scene.dim.h};
    Pico_Rel_Pos ret = {'!', {}, PICO_ANCHOR_C};
    _rel_pos(pt, &ret, &Rb);
    return ret;
}
```

`_root_rect`'s special cases (`r == NULL`, `L == R`, `L->hier.up`) collapse
because `_rect_root_to` already handles those uniformly when given the
right `fr` input.

## Steps

1.  Add `_pos_root_to` / `_pos_root_fr` in `src/geom.hc` (above the
    existing `_cv_walk_pos`).
2.  Rewrite `_cv_walk_pos` to compose the two halves.
3.  Repeat for `_dim` and `_rect` pairs.
4.  Rewire `_root_pos` / `_root_rect` to call the `_xxx_root_to` halves
    directly.
5.  Run `make int T=<cv-app>` on the visual cv apps to eyeball.
6.  Run `make test T=<cv-app>` to confirm regressions hold.

## Notes

-   Keep `_cv_walk_*`'s `L_fr == L_to` early-return at the top so the
    same-layer fast path stays free of root-walk work.
-   The shared-root `pico_assert` moves out of the down-loop into the
    composed `_cv_walk_*` so it fires once, before the second walk.
-   `chain[64]` stack array stays inside `_xxx_root_fr` — same depth bound.

## Out of scope

-   No public API change.
-   No behavior change beyond the no-op down-loop being skipped from
    `_root_pos` / `_root_rect`.
