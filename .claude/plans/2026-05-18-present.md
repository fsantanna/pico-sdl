# pico_output_present: layers parameter

Branch: TBD (proposed; affects C and Lua APIs).
Goal: add a `layers` parameter to `pico_output_present` so it can
composite the layer tree before mirroring to the framebuffer. Closes
the expert-mode footgun where an explicit `present()` silently fails
to render layer content drawn on `world`.

## 1. Motivation

Current asymmetry (`src/output.c:224-257`):

| mode       | `pico_output_present`      | composes?                       |
|------------|----------------------------|---------------------------------|
| non-expert | auto-fires after each op   | yes (`_pico_layer_draw_all`)    |
| expert     | only manual                | **no** (raw mirror only)        |

Every real caller in expert mode pairs `pico_output_draw_layers()`
with `pico_output_present()`:

- `lua/doc/rects.lua:22-23`
- `lua/doc/anims.lua:65-66`
- `lua/tst/expert.lua:17-19`
- `lua/tst/guide.lua` §8.b L331-335

The pairing is the common case; the raw-mirror variant is rare and
advanced.

## 2. Proposed API

### 2.1. C

```c
// src/pico.h
void pico_output_present (int layers);
```

- `layers=1`: invoke `_pico_layer_draw_all(&G.window.layer)` before
  the mirror step, regardless of expert mode.
- `layers=0`: skip composition; mirror `window.tex` only (current
  expert-mode behavior).
- No default value (C limitation — callers always pass explicitly).

### 2.2. Lua

```
pico.output.present ([layers: boolean])     -- defaults to true
```

- `pico.output.present()` and `pico.output.present(true)` composite +
  mirror.
- `pico.output.present(false)` mirrors only.

## 3. Resolved decisions

- [x] Q1: Non-expert mode asserts `layers=1`.
      Auto-present in non-expert always composes; `layers=0` would
      be a no-op contradiction. Assert at the public wrapper.
- [x] Q2: Keep `pico.output.draw.layers` / `pico_output_draw_layers`.
      It is the inverse of `present(false)` (compose without
      mirror), useful when sampling composed `window.tex` mid-frame
      without flushing the framebuffer. Removing it = caller churn
      for negligible API-surface saving. No rename.
- [x] Q3: Option A — extend internal signature to
      `_pico_output_present(int force, int layers)`. Replace the
      `if (!G.expert.on)` gate at `src/output.c:238` with
      `if (layers)`. Auto-present sites call `(0, 1)`; public
      wrapper calls `(1, layers)`.
      Rejected Option B (move compose to public wrapper): forces
      touching every auto-present call site for no gain.
