# pico_output_present: compose parameter

Branch: TBD (proposed; affects C and Lua APIs).
Goal: add a `compose` parameter to `pico_output_present` so it can
composite the layer tree before mirroring to the framebuffer. Closes
the expert-mode footgun where an explicit `present()` silently fails
to render layer content drawn on `world`.

## 1. Motivation

Current asymmetry (`src/output.c:224-257`):

| mode       | `pico_output_present`      | composes?                       |
|------------|----------------------------|---------------------------------|
| non-expert | auto-fires after each op   | yes (`_pico_layer_draw_all`)    |
| expert     | only manual                | **no** (raw mirror only)        |

Every real caller in expert mode pairs `pico_output_draw_layers()`
with `pico_output_present()`:

- `lua/doc/rects.lua:22-23`
- `lua/doc/anims.lua:65-66`
- `lua/tst/expert.lua:17-19`
- `lua/tst/guide.lua` §8.b L331-335

The pairing is the common case; the raw-mirror variant is rare and
advanced.

## 2. Proposed API

### 2.1. C

```c
// src/pico.h
void pico_output_present (int compose);
```

- `compose=1`: invoke `_pico_layer_draw_all(&G.window.layer)` before
  the mirror step, regardless of expert mode.
- `compose=0`: skip composition; mirror `window.tex` only (current
  expert-mode behavior).
- No default value (C limitation — callers always pass explicitly).

### 2.2. Lua

```
pico.output.present ([compose: boolean])     -- defaults to true
```

- `pico.output.present()` and `pico.output.present(true)` composite +
  mirror.
- `pico.output.present(false)` mirrors only.

## 3. Open questions

- [ ] Q1: Should non-expert mode honor `compose=0`?
      In non-expert, draws auto-present (compose=1 implicit). An
      explicit `present(false)` after manual ops could legitimately
      mean "flush framebuffer only". Probably yes for consistency.
- [ ] Q2: Deprecate or keep `pico.output.draw.layers` /
      `pico_output_draw_layers`?
      Once `present(true)` covers the common case, the standalone
      compose function is only needed when composing without
      mirroring. Keep, but rename if confusing.
- [ ] Q3: Internal `_pico_output_present(force)` already exists. Do
      we extend its signature to `(int force, int compose)`, or
      thread `compose` differently?

- Expert-mode `pico.output.present()` (default-compose) renders
  world/window children without a separate `draw.layers()` call.

  used by advanced callers.

  per snippet.
