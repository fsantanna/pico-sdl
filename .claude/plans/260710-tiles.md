## 1. Current Model

- `Tile = {w,h}`: tile size in the layer's **own texture pixels**
  (no mode tag).
- Stored per-layer in `scene.tile`; default `{0,0}` (= "no tiles").
- `'#'` mode resolves every value against the **current** layer's
  `scene.tile`:

```
pos:   raw = (x - 1 + anchor) * tile      -- 1-indexed, cell-anchored
dim:   raw = w * tile
rect:  raw = (x - 1 + anchor)*tile - anchor*(w*tile)
```

- Settable only via:

| Entry point       | Field  | Effect                                  |
|-------------------|--------|-----------------------------------------|
| `layer.empty`     | `tile` | dim Ă— tile = texture size; stores tile  |
| `set.scene`       | `tile` | stores tile (applied **before** `dim`)  |

- Tile also drives the `_show_tile` grid aid (`layer.c:136`).

## 2. Ambiguities Found

### 2.1. `dim`/`tile` convention differs between entry points

| Entry point   | `dim={'!',5,5}, tile={20,20}`  | Result          |
|---------------|--------------------------------|-----------------|
| `layer.empty` | dim Ă— tile **unconditionally** | 100x100 texture |
| `set.scene`   | `'!'` stays pixels             | 5x5 texture     |

- `layer.empty` (`mem.c:201`): multiplication happens for **any**
  dim mode when `tile` is present.
- `set.scene` (`get-set.c:330`): only `'#'` dim means tiles.
- Consequences in `layer.empty`:
    - `dim={'%',...}, tile=...`: percent of parent Ă— tile
      (surely unintended).
    - `dim={'#',...}, tile=...`: resolves against the **current**
      layer's tile (`geom.c:392`), then Ă— new tile again
      (**double multiplication**).

### 2.2. `'#'` without tile set: silent zero / division by zero

- `_raw_*` multiply by `tile=0`: silent zero-size results.
- `_rel_*` (`geom.c:525,561,601`) divide by `tile.w/h`:
  integer **division by zero** (UB/crash).
- No assert anywhere; api.md only says "(required for '#' dim)".

### 2.3. `'#'` ignores `base` (unlike `'%'`)

- `'%'` resolves against `base` (nested rect, parent, cv frame).
- `'#'` always reads `G.layer->scene.tile`, even when a `base` is
  supplied.
- Affects: `pico.cv.*`, `pico.xin.*`, `pico.vs.*`, `target` rects
  resolved in the **parent** frame, `layer.empty` dim with `up`.
- Which layer's tile *should* apply is undefined by design and
  undocumented.

### 2.4. `dim` vs `target` relation undocumented for tiles

- `dim` = texture resolution; `target` = placement in parent
  (`scene.dst`). Tile affects **only** `'#'` coordinate
  interpretation and the grid aid â€” never the dimâ†’target mapping.
- `layer.empty` defaults `dim` from the `target` table
  (`lua/pico.c:1279`): with `target={'#',...}` or `'%'` the
  resulting texture size is unclear (and then Ă— tile again).
- Nothing requires `dim % tile == 0`: fractional edge tiles are
  silently allowed.

### 2.5. `'#'` symbol overload â€” RESOLVED (260716, keep + document)

- `layer.images` `sheet={'#', w, h}` means "grid spec of the sprite
  sheet", unrelated to `scene.tile`. Same symbol, different meaning.
- Decision: **not really overloaded**. `'#'` uniformly means "count of
  grid cells along x/y"; Â§7 strengthens this (`dim`/`tile`/`pos`/`rect`
  all cell counts). Only the **grid source** differs:
    - `sheet` `'#'` â†’ the sheet image's own `wĂ—h` frame subdivision.
    - all others `'#'` â†’ the layer's `scene.tile`.
- Action: **keep `'#'`**, no rename/API break. Doc-only (5.6):
  "`'#'` always counts grid cells; in `sheet` the grid is the sprite
  sheet's own `wĂ—h` subdivision, elsewhere the layer's `scene.tile`."

### 2.6. Anchor/indexing semantics undocumented

- `'#'` is **1-indexed** (only stated in `pico.h:160` comment).
- In `'#'`, `x/y` name a **cell** and the anchor pins within the
  cell *and* within the shape; in `'!'`/`'%'`, `x/y` name a point.
- Not stated in api.md or guide.

## 7. Tile/Dim mode resolution (5.1 design)

`tile` gains a mode tag, resolved jointly with `dim`.
`P` = parent size (known).

### Definitions

Derived `T` floors to integer px (`âŚŠÂ·âŚ‹`); leftover clips (Â§5.5).

```

tile #   T = âŚŠD / tâŚ‹    (t = cells across dim)
tile %   T = âŚŠt% Â· DâŚ‹   (% of dim)

dim  !   D = d          (pixels)

```

### All 9 combinations

| dim â†“ \ tile â†’ | `!` (px)         | `#` (cells of dim) | `%` (of dim)       |
|----------------|------------------|--------------------|--------------------|
| `!` (px)       | `D=d`    `T=t`   | `D=d`    `T=D/t`   | `D=d`    `T=t%Â·D`  |
| `%` (of par)   | `D=d%Â·P` `T=t`   | `D=d%Â·P` `T=D/t`   | `D=d%Â·P` `T=t%Â·D`  |
| `#` (cells)    | `T=t`    `D=dÂ·T` | âś— circular         | âś— circular         |

### Resolution order

```
dim == '#'  â†’  T first, then D = dÂ·T   (needs tile absolute â†’ tile='!')
else        â†’  D first, then T          (tile '!','#','%' all ok)
```

### Rejects

Both cycles are dim=`#` (needs T) Ă— tile that needs D:

```
dim #, tile #  â†’  D=dÂ·T, T=D/t   â†’  D = DÂ·(d/t)     -- no anchor
dim #, tile %  â†’  D=dÂ·T, T=t%Â·D  â†’  D = DÂ·(dÂ·t%)    -- no anchor
```

Rule: **dim=`#` is valid only with tile=`!`.**

### Omitted `dim` (Â§7 D, decided 260716)

`layer.empty` with no `dim` borrows the `target`'s dim spec
(`lua/pico.c:1288`; mode included). `target` is placed in the parent
(`up`) frame, which is already the `base` in `mem.c:196`, so
target=`'%'` = "% of parent". Only the target's **dim** is reused; its
**pos** still drives placement.

- decision: **no special case** â€” `dim := target.dim`, then the Â§7
  resolver applies **unchanged** (same matrix, rejects, floor).
- target=`'#'` needs tile=`'!'` (else reject, per the matrix).
- impl (5.1): tile parse at `lua/pico.c:1299` becomes a tagged
  `Pico_Rel_Dim` (read `mode`), threaded into the resolver.
