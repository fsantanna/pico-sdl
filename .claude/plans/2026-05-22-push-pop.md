# 2026-05-22 push-pop

Expose realm scopes to user code via `pico_push` / `pico_pop`, so a
group of layers and resources can be released together when a scene
ends (typical case: menu → game transition).

## Goal

Today the realm is opened once in `pico_init(true)` and closed once
in `pico_init(false)` (`src/pico.c:81,215`). The internal
`realm_enter` / `realm_leave` (`src/realm.hc:103-120`) already
support stacked scopes via `depth`, but only one `realm_enter`
runs (`src/pico.c:84`) — there is no way for user code to bracket a
scene.

Expose two verbs:

```
pico_push (void);   // begin scope
pico_pop  (void);   // end scope — frees every entry created since
                    //              the matching push
```

No `pop(key)`. Per-key replacement keeps using realm mode `'~'`,
which already does free-old-then-alloc-new on the same entry.

## Use case (Lua)

```lua
-- menu
pico.push()
pico.layer.image("world", "bg",   "menu_bg.png")
pico.layer.text ("world", "play", 40, "PLAY", ...)
-- ... menu loop ...
pico.pop()       -- frees bg, play, every layer created in this scope

-- game
pico.push()
pico.layer.image("world", "lvl1", "lvl1.png")
pico.layer.pixmap("world", "hero", ...)
-- ... game loop ...
pico.pop()
```

Scene-replace idiom is just `pop()` followed by `push()`.

## Pre-requisite: detach-on-free

`realm_leave` frees every entry whose `depth == r->depth` in one
sweep (`src/realm.hc:107-120`). A child created in an inner scope
can have its parent in an outer scope — e.g. `pico.layer.empty
("world", "menu_bg", ...)` inside the menu's `push`. `world` lives
at depth 0; `menu_bg` lives at depth 1. When `pop` frees
`menu_bg`, `world->hier.dn.fst/lst` still names `"menu_bg"` and
sibling chains via `hier.nxt` still reference it.

The next `_pico_layer_draw_all` walk fails on
`assert(CUR != NULL)` at `src/layer.c:39`.

Fix: `_pico_mem_free_layer` (in `src/mem.c`) must, before
releasing the layer, splice itself out of `hier.up`'s child list.
Concretely, if `self->hier.up` resolves to a live parent `UP`:

- If `UP->hier.dn.fst == self->name`, advance `fst` to
  `self->hier.nxt`.
- Otherwise, walk `UP->hier.dn.fst` chain, find the predecessor
  whose `nxt == self->name`, rewrite its `nxt` to skip `self`.
- If `UP->hier.dn.lst == self->name`, set `lst` to the new tail
  (predecessor, or NULL if list emptied).

This also benefits `'~'` replace, which today already frees the
old layer entry but leaves the parent pointing at a freed
`Pico_Layer`.

## C API

```c
// src/pico.h
void pico_push (void);
void pico_pop  (void);
```

```c
// src/pico.c
void pico_push (void) {
    _pico_guard();
    realm_enter(G.realm);
}

void pico_pop (void) {
    _pico_guard();
    pico_assert(G.realm->depth > 1 && "pop without matching push");
    realm_leave(G.realm);
}
```

`depth > 1` guard: the init scope (`window`, `world`) lives at
depth 0 with `r->depth == 1` after the one-time `realm_enter`. A
user `pop` must never drop below that floor.

`pico_init(false)` is unchanged: `realm_close` walks every bucket
regardless of depth, so a user who forgot a `pop` still gets a
clean shutdown.

## Lua binding

```c
// lua/pico.c
static int l_push (lua_State* L) { pico_push(); return 0; }
static int l_pop  (lua_State* L) { pico_pop();  return 0; }

// register: pico.push, pico.pop
```

No sniffing, no varargs — zero-arg verbs.

Optional sugar in `lua/pico/init.lua`:

```lua
function pico.with (fn)
    pico.push()
    local ok, err = pcall(fn)
    pico.pop()
    if not ok then error(err, 2) end
end
```

`pico.with(function() ... end)` guarantees a `pop` even on error.
Lands as follow-up if useful; not required for v1.

## Realm modes

Keep all three modes (`'!'`, `'='`, `'~'`). They solve a different
problem (insertion-conflict resolution per key) from scope
lifetime. `'~'` is *not* redundant with `pop+push`:

- `pop+push` clears a whole scope.
- `'~'` replaces a single key in place — useful mid-scene without
  tearing down the surrounding scope.

## Tradeoff considered

| approach                | pro                              | con                            |
|-------------------------|----------------------------------|--------------------------------|
| `push`/`pop` only       | minimal API, matches realm core  | no per-key delete              |
| `push`/`pop` + `pop(k)` | granular cleanup                 | overlaps with `'~'`, more API  |
| Refcounted entries      | safe `pop(k)` for shared (`'='`) | adds a counter to every entry  |

**Pick:** `push`/`pop` only. Per-key needs are served by `'~'`,
and shared (`'='`) keys are intentionally long-lived — letting
them die with the scope is fine.

## Open questions

- Should `pico_pop` reset `G.layer` if the current target layer
  belonged to the popped scope? Otherwise the next draw call
  asserts inside `_pico_guard`/`SDL_SetRenderTarget`.
  - Proposal: snap `G.layer` to `&G.world` (the depth-0 floor)
    when its name is no longer resolvable after `realm_leave`.
- Should `pop` log/warn when entries it frees still have live
  external references via `'='` (shared)? Today there's no
  refcount, so we can't detect; deferring.
