# 2026-05 geom (cv, vs, in)

Three independent improvements to the geometry helpers.
Each section lists its failing tests first, then the change.

Builds on top of 2026-05 vs-descendant (vs now mirrors cv:
L must be cur or descendant of cur).

## 1. Bidirectional layer hierarchy

When a helper accepts two layer refs (`L1`, `L2`), allow the
relationship to point in either direction.
Today the ancestor walk is one-way:

- `cv`: target/source must be cur or **ancestor** of cur.
- `vs`: each L must be cur or **descendant** of cur (via cv).
- `in`: no layer arg today; revisit if it grows one.

Goal: if L1 is not an ancestor of L2, try L2 as ancestor of L1.
Internal: probe direction once, then reuse the existing chain walk.

Siblings (neither is ancestor of the other): **assert**.
Only direct ancestor/descendant chains are allowed.

### Affected

- `pico_cv_pos_to`, `pico_cv_pos_from`
- `pico_cv_rect_to`, `pico_cv_rect_from`
- `pico_cv_dim_to`, `pico_cv_dim_from`
- `pico_vs_pos_pos`, `pico_vs_pos_rect`,
  `pico_vs_rect_pos`, `pico_vs_rect_rect`

### Tests first

| file              | case                                           |
|-------------------|------------------------------------------------|
| tst/cv.c          | cv_pos_to with target = descendant of cur      |
| tst/cv.c          | cv_pos_from with source = descendant of cur    |
| tst/vs.c          | vs_pos_pos with L1 = ancestor (inverse of cur) |
| tst/vs.c          | vs_rect_rect with L1 and L2 siblings asserts   |
| lua/tst/cv.lua    | mirror cv.c cases                              |
| lua/tst/vs.lua    | mirror vs.c cases                              |

All should currently abort on the one-way ancestor assertion.

## 2. Output `to` argument first

Current C signature:

```c
void pico_cv_pos_to (const char* layer,
                     const Pico_Rel_Pos* fr,
                     Pico_Rel_Pos* to);
```

Proposed:

```c
void pico_cv_pos_to (Pico_Rel_Pos* to,
                     const Pico_Rel_Pos* fr,
                     const char* layer);
```

Rationale:

- Matches dst-first C convention (`memcpy`, `strcpy`, `sprintf`).
- Optional `layer` (NULL = cur) is trailing — idiomatic.
- Enables item 3 below (Lua dispatch on `to` slot).

### Affected

All `pico_cv_*_to` / `pico_cv_*_from` (6 fns) and any
`pico_in_*` that grows a `to` out-param.

### Tests first

No new test cases — the existing tst/cv.* and lua/tst/cv.lua
suites must be updated in lockstep with the C signature change.
Mark the migration done only when every call site under
`src/`, `tst/`, `lua/`, `app/` has been swept.

## 3. Lua: `'?'` mode as `to`

Today the Lua wrappers require `to` to be a pre-allocated table:

```lua
local to = {'!', anchor='NW'}
pico.cv.pos.to(to, from, 'hud')            -- after item 2
```

Goal: accept a mode-string in place of the table.
The wrapper allocates and returns a fresh `Pico_Rel_*` table.

```lua
local to = pico.cv.pos.to('!', from, 'hud')    -- abs
local to = pico.cv.rect.from('%', from, 'hud') -- pct
local to = pico.cv.dim.to('#', from, 'hud')    -- tiles
```

Dispatch in `l_cv_*`: if the `to` slot is a single-char string
in `{'!', '%', '#', 'w'}`, build and return a fresh table;
if a table, current in-place behavior; return nothing.

### Affected

- `l_cv_pos_to`, `l_cv_pos_from`
- `l_cv_rect_to`, `l_cv_rect_from`
- `l_cv_dim_to`, `l_cv_dim_from`
- (any `l_in_*` mirror, if added)

`vs` returns a boolean — unaffected.

### Tests first

| file           | case                                            |
|----------------|-------------------------------------------------|
| lua/tst/cv.lua | pos.to with mode-string `'!'`, assert returned  |
| lua/tst/cv.lua | rect.from with mode-string `'%'`                |
| lua/tst/cv.lua | dim.to with mode-string `'#'`                   |
| lua/tst/cv.lua | invalid mode-string `'z'` -> error              |

## 4. Bug: deep-descendant projection in `_vs_rect`

Discovered after items 1-3 landed. Triggered by lua tests with
`sub_sub_tight` (grandchild with explicit scene.dst).

Two distinct issues:

### 4a. Lua wrapper: `l_layer_empty` arg-5 misinterpretation

`pico.layer.empty(up, key, clear, Dim, Rect)` parsed arg 5 as Tile
(reads `w`/`h`, ignores `x`), silently dropping scene.target.

Fix: shape-discriminate trailing args 5/6 — has `x` ⇒ Rect (target);
no `x` ⇒ Tile. Error on duplicate tile/target.

Allowed call shapes now:

| dim | tile | target | call |
|---|---|---|---|
| ✓ | — | — | `(up, key, clear, Dim)` |
| ✓ | — | ✓ | `(up, key, clear, Dim, Rect)` |
| ✓ | ✓ | — | `(up, key, clear, Dim, Tile)` |
| ✓ | ✓ | ✓ | `(up, key, clear, Dim, Tile, Rect)` |
| ✓ | — | ✓ | `(up, key, clear, Rect)` (one-call) |
| ✓ | ✓ | ✓ | `(up, key, clear, Rect, Tile)` |

### 4b. C: `_vs_rect` r==NULL path for grandchildren+

When `L2`'s parent != cur, `_vs_rect` returned `L2->scene.dst` as-is.
scene.dst is in L2's **parent** frame; for grandchildren+ it must be
projected from parent up to cur via `pico_cv_rect_to`.

### Tests

| file           | case                                              |
|----------------|---------------------------------------------------|
| lua/tst/vs.lua | sub_sub_tight, explicit r_pct + default-r (added) |
| tst/vs.c       | mirror C tests for both r paths                   |

## 5. Unify cv (6 → 3 fns)

Collapse `pico_cv_*_to` and `pico_cv_*_from` into a single function
per type. Matches the `vs` shape `(L1, v1, L2, v2)`.

Style stays ptr-based here; conversion to by-value lives in the
sibling plan `2026-05-plain.md`.

```c
void pico_cv_pos (
    const char* L_to, Pico_Rel_Pos*  to,
    const char* L_fr, const Pico_Rel_Pos*  fr
);
void pico_cv_rect (
    const char* L_to, Pico_Rel_Rect* to,
    const char* L_fr, const Pico_Rel_Rect* fr
);
void pico_cv_dim (
    const char* L_to, Pico_Rel_Dim*  to,
    const char* L_fr, const Pico_Rel_Dim*  fr
);
```

- `L_*` `NULL` ⇒ cur.
- `to->mode` / `.anchor` are the output template; `x/y/w/h` filled.
- Internal dispatch (3 cases):
  - `fr` is cur ⇒ direct walk to `L_to` (existing `_to` body).
  - `to` is cur ⇒ direct walk from `L_fr` (existing `_from` body).
  - neither ⇒ two-step via cur (siblings/cousins).

Siblings now work via cur; **drop the sibling-assert rule from
item 1**.

### Old → new mapping

| old | new |
|---|---|
| `pico_cv_pos_to(&to, &fr, L)`   | `pico_cv_pos(L, &to, NULL, &fr)` |
| `pico_cv_pos_from(&to, &fr, L)` | `pico_cv_pos(NULL, &to, L, &fr)` |
| `pico_cv_rect_to/from` | `pico_cv_rect(L_to, &to, L_fr, &fr)` |
| `pico_cv_dim_to/from`  | `pico_cv_dim(L_to, &to, L_fr, &fr)` |

### Lua wrappers

Collapse `pico.cv.pos.to/from` → `pico.cv.pos` (and similarly for
rect, dim):

```lua
-- table form (in-place fill):
pico.cv.pos (L_to, to_table, L_fr, from_table)

-- mode-string form (allocate + return):
local p = pico.cv.pos (L_to, '!', L_fr, from_table)

-- L_* may be nil (== cur):
local p = pico.cv.pos (nil,   '!', 'hud', src)   -- was pos.from
local p = pico.cv.pos ('hud', '!', nil,   src)   -- was pos.to
```

### Affected

- `src/pico.h` (6 prototypes → 3)
- `src/geom.hc` (refactor walks; new 3 public entry points)
- `src/pico.c` (internal callers: `pico_set_mouse`,

- `lua/pico.c` (6 `l_cv_*` → 3 unified wrappers)
- `lua/doc/api.md` (`pico.cv` section rewrite)
