# Plan: pico.get.{image,text,video} — Inverted, Flexible Signatures

Invert the data-first signatures of `pico.get.image`, `pico.get.text`,
and `pico.get.video` so the "thing-to-fill / target" comes **before**
the required data, matching `get.mouse` / `set.mouse` / `cv.*`.

For `image` and `video`, Lua callers may pass `string` (mode), `table`
(dim/rect), `nil`, or nothing in the optional slot. For `text`, the
target dim stays required (it carries the input font-size `h`), but
its position moves before the text.

Order of work: **image first**, then `text`, then `video`.

## Current

| api   | layer | signature                                                                |
|-------|-------|--------------------------------------------------------------------------|
| image | C     | `Pico_Abs_Dim pico_get_image (const char* path, Pico_Rel_Dim* rel)`      |
| image | Lua   | `pico.get.image (path: string [, dim: Dim]) -> Dim`                      |
| text  | C     | `Pico_Abs_Dim pico_get_text (const char* text, Pico_Rel_Dim* dim)`       |
| text  | C     | `pico_get_text_mode (int mode, const char* key, const char* text, Pico_Rel_Dim* dim)` |
| text  | Lua   | `pico.get.text (text: string, dim: Dim) -> Dim`                          |
| video | C     | `Pico_Video pico_get_video (const char* path, Pico_Rel_Rect* rect)`      |
| video | Lua   | `pico.get.video (path: string [, rect: Rect]) -> Video`                  |

## Proposed

| api   | layer | signature                                                                |
|-------|-------|--------------------------------------------------------------------------|
| image | C     | `Pico_Abs_Dim pico_get_image (Pico_Rel_Dim* rel, const char* path)`      |
| image | Lua   | `pico.get.image ([dim: string\|Dim?,] path: string) -> Dim`              |
| text  | C     | `Pico_Abs_Dim pico_get_text (Pico_Rel_Dim* dim, const char* text)`       |
| text  | C     | `pico_get_text_mode (int mode, const char* key, Pico_Rel_Dim* dim, const char* text)` |
| text  | Lua   | `pico.get.text (dim: string\|Dim, text: string) -> Dim`                  |
| video | C     | `Pico_Video pico_get_video (Pico_Rel_Rect* rect, const char* path)`      |
| video | Lua   | `pico.get.video ([rect: Rect?,] path: string) -> Video`                  |

Default mode stays `'!'` for image; `'='` for text. Video has no
string-mode shorthand (see Return Types below).

### Accepted Lua call forms — `image`

| call                                | dim built / used        | returns  |
|-------------------------------------|-------------------------|----------|
| `pico.get.image(path)`              | `{'!', w=0, h=0}`       | new tbl  |
| `pico.get.image(nil, path)`         | `{'!', w=0, h=0}`       | new tbl  |
| `pico.get.image('!', path)`         | `{'!', w=0, h=0}`       | new tbl  |
| `pico.get.image('%', path)`         | `{'%', w=0, h=0}`       | new tbl  |
| `pico.get.image('#', path)`         | `{'#', w=0, h=0}`       | new tbl  |
| `pico.get.image({'%'}, path)`       | given table, w/h filled | same tbl |
| `pico.get.image({'%',w=…}, path)`   | given table, h filled   | same tbl |

### Accepted Lua call forms — `text`

`dim` is **required as a table** — `h` must be set (input font size).
No string-mode shorthand: it would build `{mode, w=0, h=0}` which
asserts on non-empty text since `h==0`.

| call                                       | dim built / used        |
|--------------------------------------------|-------------------------|
| `pico.get.text({'=', h=20}, text)`         | given table, w filled   |
| `pico.get.text({'!', h=20}, text)`         | given table, w filled   |
| `pico.get.text({'%', h=20}, text)`         | given table, w filled   |

### Accepted Lua call forms — `video`

No string-mode shorthand: a `Rect` carries position too, so the
realm-mode characters don't have a meaningful interpretation.

| call                                    | rect built / used       | returns  |
|-----------------------------------------|-------------------------|----------|
| `pico.get.video(path)`                  | `NULL`                  | new Video |
| `pico.get.video(nil, path)`             | `NULL`                  | new Video |
| `pico.get.video({'%'}, path)`           | given table, w/h filled | new Video |
| `pico.get.video({'%',w=…,h=…}, path)`   | given table             | new Video |

## Comparison with sibling APIs

| API                                                      | pattern                                |
|----------------------------------------------------------|----------------------------------------|
| `pico.get.mouse ([layer?,] mode\|pos)`                   | context first, value last; supports string-mode or table via shift trick |
| `pico.set.mouse ([layer?,] pos)`                         | same shape                             |
| `pico.cv.pos (L_to?, to, L_fr?, fr)`                     | target before source                   |
| `pico.cv.dim`, `pico.cv.rect`                            | same                                   |
| `pico.get.text (text, dim)`                              | data first — **inconsistent**, candidate for later sibling change |
| `pico.get.video (path, [rect])`                          | data first — **inconsistent**, candidate for later sibling change |

After change, `pico.get.image` matches the `get.mouse` / `set.mouse`
shape: optional `(mode|tbl|nil)` first, then the required positional
value.

## Return Types

### C layer — unchanged

| fn                | returns          | mutates rel/rect?                          | NULL rel/rect? |
|-------------------|------------------|--------------------------------------------|----------------|
| `pico_get_image`  | `Pico_Abs_Dim`   | yes — auto-fills 0 axes from natural size  | OK             |
| `pico_get_text`   | `Pico_Abs_Dim`   | yes — fills `w` when input is 0            | asserted       |
| `pico_get_video`  | `Pico_Video`     | yes — fills 0 axes from natural size       | OK             |

The abs return is what the function actually computes; the mutated
rel/rect carries the realm-converted (zero-axes-filled) result.

### Lua layer — deliberate asymmetry

| api    | Lua returns                          | why                                                    |
|--------|--------------------------------------|--------------------------------------------------------|
| image  | the **rel dim** (mutated or built)   | string/nil callers hold no table; return is their only handle on the realm result |
| text   | fresh abs `{w, h}`                   | text's primary question is "pixel width" — caller wants pixels |
| video  | full `Video` struct                  | orthogonal — Video already carries the dim             |

Image returning the rel (not the abs) is intentional, not an
oversight: it is the only return shape that stays useful across all
three input forms (`'%'` / `nil` / table) for that API.

## Implementation Notes

### Step 1 — `image` (do first, in isolation)

**C layer**

- `src/pico.h`     — reorder `pico_get_image` proto + doc comments
- `src/get-set.c`  — reorder impl (body identical, only params swap)
- in-tree callers  — `grep pico_get_image` and swap args at each site

**Lua layer** (`lua/pico.c::l_get_image`)

Use the shift trick (mirror of `L_layer_opt_mode` at `lua/pico.c:80`):

1. If `lua_gettop(L) == 1`, the lone arg is the `path`. Push `nil`
   and `lua_insert(L, 1)` to shift it into slot 2.
   Stack is now `dim_or_nil_or_mode | path`.

2. Branch on slot-1 type:
    - `LUA_TNIL`    → build default dim `{'!', w=0, h=0}` at slot 1
    - `LUA_TSTRING` → verify it is a mode via `C_mode_s_opt`,
       then build `{mode, w=0, h=0}` at slot 1
    - `LUA_TTABLE`  → leave as-is (fill in place)
    - else          → `luaL_typeerror`

3. Call the existing helper `L_image_get_dim(L, 1, path)`
   (note slot **1**, not 2, since args inverted).

4. Return 1 — the slot-1 dim (the same table the caller passed,
   or the freshly built one).

**Docs**

Update `lua/doc/api.md` `pico.get.image` block (signature + forms).

### Step 2 — `text`

**C layer**

- `src/pico.h`     — reorder both `pico_get_text` and
                     `pico_get_text_mode` protos + docs
- `src/get-set.c`  — reorder both impls
- in-tree callers  — `grep pico_get_text` and swap

**Lua layer** (`lua/pico.c::l_get_text`)

`dim` stays required and must be a table (no string/nil form).

1. `luaL_checktype(L, 1, LUA_TTABLE)`
2. `luaL_checkstring(L, 2)` for text.
3. Call `pico_get_text(&dim, text)` with the inverted order.
4. Write back `w`/`h` to slot 1; return a fresh abs `{w, h}` table.

**Docs**

Update `lua/doc/api.md` `pico.get.text` block (signature + forms).

### Step 3 — `video`

**C layer**

- `src/pico.h`     — reorder `pico_get_video` proto + doc
- `src/get-set.c`  — reorder impl
- in-tree callers  — `grep pico_get_video` and swap

**Lua layer** (`lua/pico.c::l_get_video`)

Mirror the image shift trick (push `nil` + `lua_insert` when only one
arg). Branches on slot 1:

- `LUA_TNIL`    → pass `NULL` rect to C
- `LUA_TTABLE`  → use as rect (mutated in place by C)
- else          → `luaL_typeerror`

No string-mode branch. Then `pico_get_video(rect, path)`. Return the
full `Video` table (unchanged from current behavior).

**Docs**

Update `lua/doc/api.md` `pico.get.video` block (signature + forms).
