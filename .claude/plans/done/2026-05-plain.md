# 2026-05 plain (by-value public API)

Convert pointer args / out-params to plain values across the
public C and Lua API. Pure ptr→plain sweep; no semantic /
naming changes.

Prereq: `2026-05-geom.md` §5 (cv unify 6 → 3 fns, still ptr-style).
This plan converts those 3 (and the rest of the eligible surface)
to by-value.

## Context

The public surface today mixes ptr/by-value style: drawing
primitives, setters, getters take pointers needlessly. Goal: every
public C function takes plain values and returns plain values; no
in/out pointer params (arrays excepted).

Functions where `NULL` carries a load-bearing default semantic
(`vs`, `screenshot`, `draw_layer`, `get_mouse`) stay pointer-based —
converting them would require a `PICO_REL_NONE` magic-mode sentinel,
which is worse than the current pointer. Likewise the input-event
idiom (`Event* + dt`) and the large `Pico_Layer_Scene` getter stay.

## 1. Eligible conversions (25 fns)

### 1.1 cv (3 fns; assumes geom §5 has landed) — **WON'T FIX**

Lowest leverage of the six phases. The existing `(to*, fr*)` shape
already works (no bug, no awkward template-mutation). By-value would
either pass a half-initialized struct (option A) or introduce
`Pico_Anchor` into the public signature (option B). Most churn for
marginal API cleanup. Keep ptr-style for cv.

```c
Pico_Rel_Pos  pico_cv_pos  (const char* L_to, Pico_Rel_Pos  to_tmpl,
                            const char* L_fr, Pico_Rel_Pos  fr);
Pico_Rel_Rect pico_cv_rect (const char* L_to, Pico_Rel_Rect to_tmpl,
                            const char* L_fr, Pico_Rel_Rect fr);
Pico_Rel_Dim  pico_cv_dim  (const char* L_to, Pico_Rel_Dim  to_tmpl,
                            const char* L_fr, Pico_Rel_Dim  fr);
```

`to_tmpl.mode/anchor` read; `x/y/w/h` ignored. Returns projected.

### 1.2 in (3 fns)

```c
Pico_Rel_Rect pico_in_rect (Pico_Rel_Rect out, Pico_Rel_Rect in);
Pico_Rel_Pos  pico_in_pos  (Pico_Rel_Rect out, Pico_Rel_Pos  in);
Pico_Rel_Dim  pico_in_dim  (Pico_Rel_Rect out, Pico_Rel_Dim  in);
```

### 1.3 output draw (10 fns)

```c
void pico_output_draw_image     (const char* path,                  Pico_Rel_Rect rect);
void pico_output_draw_line      (Pico_Rel_Pos p1,                   Pico_Rel_Pos  p2);
void pico_output_draw_pixel     (Pico_Rel_Pos pos);
void pico_output_draw_rect      (Pico_Rel_Rect rect);
void pico_output_draw_tri       (Pico_Rel_Pos p1, Pico_Rel_Pos p2,  Pico_Rel_Pos p3);
void pico_output_draw_oval      (Pico_Rel_Rect rect);
void pico_output_draw_text      (const char* text,                  Pico_Rel_Rect rect);

                                 const char* text,                  Pico_Rel_Rect rect);
int  pico_output_draw_video     (const char* path,                  Pico_Rel_Rect rect);
```

`pico_output_draw_layer` and `pico_output_screenshot` keep `Rect*`
(NULL default useful). Pixmap/pixels/poly stay pointer-based (arrays).

### 1.4 setters (3 fns)

```c
void pico_set_dim       (Pico_Rel_Dim dim);
void pico_set_scene_dim (Pico_Rel_Dim dim);
void pico_set_mouse     (Pico_Rel_Pos pos);
```

### 1.5 mixed-return getters (4 fns) — **WON'T FIX**

The in-place template mutation is ugly but works; introducing two
new `_Pair` types and rewriting ~10-15 call sites for cosmetic API
cleanup isn't worth the churn.

Eliminate in-place template mutation; new paired types.

```c
typedef struct {
    Pico_Rel_Dim rel;     // resolved (w/h filled if 0 on input)
    Pico_Abs_Dim abs;     // absolute pixel size
} Pico_Dim_Pair;

Pico_Dim_Pair pico_get_image     (const char* path, Pico_Rel_Dim dim);
Pico_Dim_Pair pico_get_text      (const char* text, Pico_Rel_Dim dim);
Pico_Dim_Pair pico_get_text_mode (int mode, const char* key,
                                  const char* text, Pico_Rel_Dim dim);

typedef struct {
    Pico_Video    video;
    Pico_Rel_Rect rel;
} Pico_Video_Pair;

Pico_Video_Pair pico_get_video (const char* path, Pico_Rel_Rect rect);
```

### 1.6 state getters (3 fns)

```c
Pico_Layer_Pencil pico_get_pencil (void);
Pico_Layer_Effect pico_get_effect (void);
Pico_Window       pico_get_window (void);
```

`pico_get_scene` stays out-ptr (large struct).

## 2. Stay as-is

| function | reason |
|---|---|
| `pico_vs_*` (4 fns)         | NULL=bounds load-bearing |
| `pico_output_screenshot`    | NULL=full screen useful default |
| `pico_output_draw_layer`    | NULL=full layer useful default |
| `pico_get_scene`            | large struct copy |
| `pico_get_mouse`            | NULL=no rect common |
| `pico_input_event*`         | `(Event*, type) → dt` is the C idiom |
| `pico_get_expert(int* fps)` | optional primitive out |
| `draw_pixmap/pixels/poly`   | array args |

## 3. Old → new mapping (sweep table)

| old | new |
|---|---|
| `pico_cv_pos(L_to, &to, L_fr, &fr)`  | `pico_cv_pos(L_to, to_tmpl, L_fr, fr) → Pos` |
| `pico_cv_rect/dim`                   | same shape |
| `pico_in_*(&out, &in)`               | `pico_in_*(out, in)` |
| `pico_output_draw_*(&v)`             | `pico_output_draw_*(v)` (except layer) |
| `pico_set_{dim,scene_dim,mouse}(&v)` | by-value |
| `pico_get_image(path, &dim) → Abs`   | `pico_get_image(path, dim) → Dim_Pair` |
| `pico_get_text*(text, &dim) → Abs`   | `… → Dim_Pair` |
| `pico_get_video(path, &rect) → Video`| `… → Video_Pair` |
| `pico_get_{pencil,effect,window}(&out)` | return by-value |

## 4. Lua wrappers

Mechanical: flip `pico_X(&v)` → `pico_X(v)` and `pico_get_X(&out);
use out` → `auto v = pico_get_X();`.

cv mode-string return form unchanged (already in geom item 3, kept
in geom §5). With by-value C signature, the C side returns the
projected value; Lua wrapper pushes it directly without the
out-pointer fill step.

`l_get_image/text/video` read from `Pair.rel` / `Pair.abs` instead
of mutating the input table — caller still passes a Dim/Rect to
specify input mode and gets results back in the Lua return value.

## 5. Phases (each PR-sized, tests-first)

1. **cv by-value** — 3 fns (after geom §5 lands).
2. **in by-value** — 3 fns mechanical.
3. **output draw by-value** — 10 fns.
4. **setters by-value** — 3 fns.
5. **mixed-return getters** — 4 fns + 2 `_Pair` types.
6. **state getters** — 3 fns.

Each phase: tests-first → C signature change → call-site sweep
(`src/`, `tst/`, `lua/`, `app/`) → Lua wrapper → docs → green.
