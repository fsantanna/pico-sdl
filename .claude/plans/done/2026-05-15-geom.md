## Bug: cv aspect-fill blind spot

`_cv_walk_*` resolves each layer's `scene.dst` via
`_sdl_rect(L->scene.dst, &Pb, NULL)` — third arg hard-coded `NULL`.
When `scene.dst.w == 0` or `.h == 0`, `_f_rat` returns the unfilled
zero, collapsing the projection scale to 0 and losing coordinate
info.

Repro: see `tst/cv.c` "aspect-fill mid layer" section.
Mid 50×50 under world 100×100 with `scene.dst = {'!', 0,0,0,100}`.
`(10,10) mid → world` should give `(20,20)` but cv returns `(0,20)`.

## Eager normalization + attachment rule (implemented)

`pico_set_scene_dst` now **asserts** `L->hier.up != NULL` for
all modes. Rationale: `scene.dst` is the layer's placement in
its parent during auto-composite. A parentless layer is never
auto-composited, so storing a `scene.dst` is meaningless.

For explicit, draw-time placement of a parentless "sprite" layer,
use `pico_output_draw_layer(name, &rect)` — that path resolves
against current `G.layer` at draw time and accepts any mode
(including `'%'`) without a parent.

When dst.w/h is zero, eager-resolve via
`_sdl_rect(dst, &Pb, &L->scene.dim)` with `Pb = {0,0,
P.dim.w, P.dim.h}` — works for all modes (Pb is offset for
`'!'`/`'#'`; base for `'%'`). Stored as concrete `'!'`
NW-anchored rect, idempotent under later
`_sdl_rect(_, NULL, NULL)` calls.

Ripple after eager:
- `_pico_output_draw_layer`:296-298 — dead, remove
- `_cv_walk_*` — works as-is (no zero w/h ever seen)
- `_root_rect` r==NULL else-arm — one-liner
  `pico_cv_rect(R->name, &ret, L->hier.up, &L->scene.dst)`
- `pico_set_scene_dim` — does NOT re-resolve dst (option B)
- API contract: callers must set `dim` before any aspect-fill `dst`
