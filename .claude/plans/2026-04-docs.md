    ### A. API doc bugs (api.md vs code)

    - A1. **`pico.get.scene` field name wrong**
          api.md:93 says `keep: boolean`; pico.c:879-880 returns
          `clear: boolean`.
    - A2. **`pico.set.scene` cfg field name wrong**
          api.md:216 lists `[keep: boolean]`; pico.c:1044-1048
          accepts `clear: boolean`.
    - A3. **Event tag string mismatch**
          api.md:21,108 use `'win.resize'`; pico.c:1341,1798 emit
          and filter `'window.resize'`. Affects both Event type
          and filter list.
    - A4. **`pico.set.layer` return undocumented**
          api.md:220-222 shows no return. Code returns the previous
          layer name; tests (`tst/window.lua:15`,
          `tst/layer-empty-tile.lua`) rely on it via
          `local old = pico.set.layer("window")`.
    - A5. **`pico.layer.image/video` omitted-key cache name**
          api.md:131-132,156-157 claim `"/image/<path>"` /
          `"/video/<path>"`; code uses `<path>` directly
          (l_layer_image:1273-1275, l_layer_video:1335-1337).
          **RESOLVED**: code wins ‚Äî fix api.md.
    - A6. **Realm-mode prefix list incomplete**
          api.md:114 lists `'!' '%' '#' '='`. C_mode_opt
          (pico.c:70) also accepts `'~'`; `'%'` and `'#'` are
          dim-modes, not realm-modes. Confused docs ‚Äî separate
          "layer realm" prefix from "dim" prefix.
    - A7. **`pico.set.window` `dim` field**
          api.md:218-219 lists only `{fullscreen,show,title}`.
          **RESOLVED**: code rejects `dim` ‚Äî l_set_window
          (pico.c:1129-1158) only reads `fullscreen`/`show`/`title`.
          api.md is correct; guide.md ¬ß2.2 and lua/README.md
          "Hello World" are stale ‚Äî fix both.
    - A8. **`pico.cv.*` mutation contract**
          api.md:54-67 shows `(layer, fr, to)` with no return.
          Code mutates `to` in place and returns 0 ‚Äî add explicit
          note.
    - A9. **`pico.set.expert` no-arg fps default**
          api.md:208-210 omits behavior when `fps` is omitted;
          code defaults to `0` (immediate). Clarify.
    - A10. **Operations alphabetical order broken**
          api.md "Operations" section advertises alphabetical,
          but ordering is: color, cv, get, init, input, layer,
          output, quit, set, vs ‚Äî so `cv` precedes `get` (ok),
          but within `get.*` the order is pencil, image, keyboard,
          layer, mouse, now, text, video, effect, scene, window ‚Äî
          NOT alphabetical. Same issue in `set.*` and
          `output.draw.*` (e.g., image/layer/layers/line/oval...).
    - A11. **`pico.xin.*` (in/rect/pos/dim) undocumented**
          Registered as a sub-module in `luaopen_pico_native`
          (lua/pico.c:1776-1777) ‚Äî wraps `pico_in_rect/pos/dim`
          C functions (compose a child rect/pos/dim onto a
          parent rect). Completely absent from api.md.
          Naming: `in` is a Lua reserved word, so the Lua-side
          module is `xin`. Add a section to api.md.
    - A12. **Dead `'w'` branch in `C_mode_t`**
          lua/pico.c:53 accepts `'w'` as a valid mode for
          Pos/Rect/Dim tables, but the error message lists only
          `'!'`/`'%'`/`'#'`, `C_mode_s_opt` (pico.c:68) rejects
          `'w'`, and the `pico.h` mode comment (line 54-59) does
          not list it. Leftover from the `'w'`-mode removal.
          Code-only cleanup ‚Äî out of scope for this docs branch;
          flag as follow-up.

    ### F. Post-audit re-review (2026-05-18)

    Re-checked api.md vs lua/pico.c and src/pico.h for current
    state. Mismatches still present in api.md:

    - F-M1. **cv mode-string default anchor**
            api.md:58 says "anchor defaults to `'NW'`
            (Pos/Rect only)"; code uses `PICO_ANCHOR_C` (center)
            at `l_cv_pos`:481 and `l_cv_rect`:517.
    - F-M2. **layer creator realm prefixes wrong**
            api.md:129 lists `'!' '%' '#' '='`; `L_realm_opt`
            (lua/pico.c:1170) only accepts `'!' '=' '~'`.
            `'%'` and `'#'` are dim modes, not realm modes;
            `'~'` is missing. Same as audit A6 but for api.md.
    - F-M3. **Mouse Event missing anchor field**
            api.md:24-26 shows event `{tag, '!', x, y, l, r, m}`;
            `L_set_mouse`:386 also pushes `anchor: Anchor`.
    - F-M4. **`layer.image` / `layer.video` rect-without-key**
            api.md:147,172 claim "rect is not allowed in that
            form" (no-key); code (`l_layer_image`:1278,
            `l_layer_video`:1340) supports rect after the
            `me = path` fallback.
    - F-M5. **`layer.empty` signature incomplete**
            api.md:135-142 shows two forms;
            code supports 4:
                (up, key, clear, dim)
                (up, key, clear, rect)              <- dim+dst
                (up, key, clear, dim, tile)
                (up, key, clear, dim, tile, rect)   <- +dst
            The `(up, key, clear, rect [, tile])` form is
            misleading ‚Äî when tile is added, rect's pos is
            silently ignored (read as Dim only).
    - F-M6. **`image`/`video` default realm mode asymmetric**
            With explicit `key`: default `'!'` (exclusive).
            Without `key`:       default `'='` (shared/cached).
            api.md does not state this. l_layer_image:1259-1262.
    - F-M7. **`layer.sub` ordering**
            api.md:173 places `sub` after `video`; breaks
            alphabetical (should be between pixmap and text).
            Same family as audit A10.
    - F-M8. **`set.layer` return type**
            api.md:243 says `-> string`; `l_set_layer`:993
            pushes `nil` when `G.layer==NULL` (pre-init only).
            Strictly `string?`. Low priority.
