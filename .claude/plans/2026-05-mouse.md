# mouse API symmetry + anchor support

## Background

`pico_set_mouse` / `pico_get_mouse` were asymmetric:

```c
void       pico_set_mouse (const char* layer, Pico_Rel_Pos pos);
Pico_Mouse pico_get_mouse (const char* layer, char mode);
```

`set` takes a full Pos (mode + anchor).
`get` only takes mode; anchor was hardcoded `PICO_ANCHOR_NW` inside.
This left no way for callers to request cell-center semantics on get.

## Decision: option A (full symmetry, C anchor default everywhere)

### C signature

```c
- Pico_Mouse pico_get_mouse (const char* layer, char mode);
+ Pico_Mouse pico_get_mouse (const char* layer, Pico_Rel_Pos* pos);
```

- `pos->mode` + `pos->anchor` are read on input.
- `pos->x` / `pos->y` are filled on output (in place).
- Function also returns full `Pico_Mouse` with mode + anchor + x/y +
  buttons.
- Caller can use either form (pointer write-back OR return value).

### `Pico_Mouse` struct (events.h)

```c
typedef struct {
    char mode;
    float x, y;
    Pico_Anchor anchor;     // NEW
    unsigned left   : 1;
    unsigned right  : 1;
    unsigned middle : 1;
} Pico_Mouse;
```

Requires `events.h` to `#include "anchors.h"`.

### Lua

```lua
pico.get.mouse ([layer,] mode_or_pos)
```

- mode string `'!'/'%'/'#'` → binding builds
  `Pico_Rel_Pos{mode, anchor=C}` (C is the Lua-wide default).
- table `{mode, anchor=...}` → used directly as template;
  anchor defaults to C via existing `C_anchor` helper.

Returned Lua table now also has an `anchor` field
(mirrors the struct).

## Breaking change note

Defaulting to **C** for the Lua mode-only form shifts every existing
`pico.get.mouse('!')` result by `+0.5` per axis (cell-center echo).
Existing tests `assert(pos.x==123 and pos.y==456)` become
`pos.x==123.5 and pos.y==456.5` — or rewrite the call to
`pico.get.mouse({'!', anchor='NW'})`.

This matches the rest of the codebase, where anchor defaults to C
(cv, vs, layer creators, etc.).
