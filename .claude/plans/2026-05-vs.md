# 2026-05-vs — flexible arg parsing for `pico.vs.*` + new `rect_pos`

## Context

`pico.vs.pos_pos`, `pos_rect`, `rect_rect` today expose
hand-rolled 2-arg / 4-arg forms with brittle dispatch
(see `lua/pico.c:545-644`).
Each must be called either with all positional args or
with explicit nils — many useful shortenings are illegal.

Goal:

- Accept *every* arg combination — with or without
  explicit nils.
- A side whose `L` and value are both absent /
  nil defaults to "cur's bounds" (only meaningful for
  rect sides — see restrictions).
- Add `pico.vs.rect_pos` for symmetry with `pos_rect`.

## Dispatch algorithm (Lua side)

Canonical 4-slot template, same for all four functions:

```
slots:    [ L1 , x1 , L2 , x2 ]
types:    [ str, tbl, str, tbl ]
```

`x1` / `x2` are Pos or Rect tables depending on function:

| fn        | x1   | x2   |
|-----------|------|------|
| pos_pos   | Pos  | Pos  |
| pos_rect  | Pos  | Rect |
| rect_pos  | Rect | Pos  |
| rect_rect | Rect | Rect |

Walk slots left-to-right while walking the args list:

```
ai = 1
for si = 1..4:
    if ai > nargs:
        slot[si] = nil
    else if arg[ai] is nil:
        slot[si] = nil ; ai += 1          -- consume explicit nil
    else if type(arg[ai]) matches types[si]:
        slot[si] = arg[ai] ; ai += 1      -- consume
    else:
        slot[si] = nil                    -- leave empty, don't consume
if ai <= nargs: error "too many / unmatched args"
```

Restrictions enforced after the walk:

| fn        | required        |
|-----------|-----------------|
| pos_pos   | x1 ≠ nil, x2 ≠ nil |
| pos_rect  | x1 ≠ nil           |
| rect_pos  | x2 ≠ nil           |
| rect_rect | — (none)           |

Rationale: a Pos has no sensible default; a Rect
defaults to cur's bounds.

## Helper sketch

In `lua/pico.c` introduce one helper used by all four wrappers:

```c
// types[i] = LUA_TSTRING or LUA_TTABLE
// out[4]   = arg index (1..nargs) or 0 for "absent"
static void vs_parse (lua_State* L, const int types[4], int out[4]) {
    int n = lua_gettop(L), ai = 1;
    for (int si = 0; si < 4; si++) {
        out[si] = 0;
        if (ai > n) continue;
        if (lua_isnil(L, ai))           { ai++; continue; }
        if (lua_type(L, ai) == types[si]) { out[si] = ai; ai++; }
        // else: leave si empty, don't advance ai
    }
    if (ai <= n) luaL_error(L, "vs: unmatched argument at position %d", ai);
}
```

Each wrapper then calls `vs_parse` with its type pattern
and reads `out[]` to build the C call.
